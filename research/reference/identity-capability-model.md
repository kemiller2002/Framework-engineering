# Identity-Capability Model

Status: Working draft

## Purpose

This reference defines the FE-008 distinction between artifact identity and artifact capabilities.

## Core Distinctions

Identity answers:
"What is this?"

Capabilities answer:
"What can it do?"

Composition answers:
"How does it work?"

Domain answers:
"Where is it applied?"

## Rules

- Identity is determined by dominant purpose.
- Capabilities may overlap across different identities.
- Composition should be recorded only when reasoning structure exists.
- Framework Grammar and Primitive Cognitive Operations belong under composition.
- Identity and capabilities should not be conflated.

## Use

The Identity-Capability Model supports:

- KACS
- blind review
- corpus schema
- artifact boundary analysis
- future FE-008 measurement work
