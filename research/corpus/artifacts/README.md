# Artifact Folders

Status: Working draft

Each subfolder represents one characterized knowledge artifact.

Folder naming:
Use stable snake_case IDs.

Examples:

- swot
- five_whys
- decision_matrix
- ooda_loop
- cynefin
- balanced_scorecard

Each artifact folder should contain:

README.md:
Brief artifact summary and current characterization status.

kacs-v1.yaml:
Current characterization using FEMS-1.

evidence.md:
Evidence supporting identity, capabilities, composition, and lifecycle.

notes.md:
Analyst notes, ambiguities, and open questions.

characterization-history.md:
Version history and rationale for changes.
