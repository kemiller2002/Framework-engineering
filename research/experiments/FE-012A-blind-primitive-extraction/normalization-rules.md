# FE-012A Normalization Rules

## Initial Rules

- Inspect / examine / review / scan may normalize to `Observe` if the action acquires information.
- Sort / group / categorize may normalize to `Classify` if the action organizes information.
- Judge / assess / rate may normalize to `Evaluate` if the action applies criteria.
- Choose / select / commit may normalize to `Decide` if the action commits to an option.
- Implement / execute / intervene may normalize to `Act` if the action changes the system.
- Check / confirm / test may normalize to `Verify` if the action tests whether a claim or result holds.
- Revisit / update / reconsider may normalize to `Reassess` if the action reopens a prior judgment under new evidence.
- Learn / review outcomes may normalize to `Reflect` if the action extracts lessons from experience.

## Rules

- Do not merge verbs only because they sound similar.
- Merge only when the reasoning function is equivalent.
- Document every merge.
- Preserve disagreements.
- Prefer adding `candidate primitive` over forced normalization when uncertain.

## Caution

These rules are a starting reference only.

The experiment should test whether the vocabulary stabilizes, not assume that the seed vocabulary is final.
