# FE-012B Protocol

## Primitive Source

Use the frozen FE-012A primitive vocabulary exactly as the starting synthesis set:

- Bound
- Observe
- Classify
- Interpret
- Explain
- Generate
- Evaluate
- Decide
- Act
- Verify
- Reflect
- Reassess
- Communicate
- Prioritize
- Compare

No new primitives may be introduced during synthesis.

## Problem Set

- Use at least ten intentionally unrelated problems.
- Avoid Clarity, EDF, and any existing framework examples.
- Do not search for existing methodologies.

## Synthesis Process

For every problem:

1. Define scope.
2. Select only primitives.
3. Compose a reasoning sequence.
4. Document rationale.
5. Record uncertainty.
6. Record missing primitives if composition becomes impossible.

## Reviewer Evaluation

Independent reviewer evaluates:

- Internal coherence
- Completeness
- Simplicity
- Missing reasoning steps
- Primitive sufficiency
- Unnecessary primitives

Ratings:

- Strong
- Moderate
- Weak
- Unclear

No numeric scores.

## Primary Metrics

- primitive reuse
- primitive omissions
- missing primitive requests
- synthesis failures
- reviewer recognizability
- reviewer coherence

## Most Important Metric

Primitive Sufficiency Index

Definition:

Percentage of synthesis attempts completed without requiring a new primitive.

This metric is experimental.

Do not publish it as validated.

## Interpretation Constraint

Do not claim generated frameworks are useful.

Only claim whether primitive composition appears expressive enough to represent coherent procedural reasoning.
