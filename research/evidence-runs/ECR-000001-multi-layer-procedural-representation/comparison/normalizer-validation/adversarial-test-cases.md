# Adversarial Test Cases

Use `adversarial-test-cases.json` as the script input source.

Each case includes:
- `case_id`
- `phrase_a`
- `phrase_b`
- `expected_relationship`
- `rationale`

Relationships:
- equivalent
- related_but_distinct
- opposite
- unrelated
- hierarchical_parent_child
- unclear

Covered cases:
- termination vs decision
- generation vs prioritization
- gathering vs evaluation
- comparison vs verification
- entry identification vs downstream solving
- reassessment vs reflection
- coordination vs communication
- hierarchy vs procedure
- evidence sufficiency vs pragmatic stopping
- execution vs validation
- equivalent entry, generation, acquisition, loop, allocation, and delivery cases
