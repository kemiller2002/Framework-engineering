# Framework Engineering Measurement Standard 1

Status: Foundation Candidate

## Purpose

Provide a stable measurement instrument for Knowledge Artifact characterization.

## Measurement Dimensions

### 1. Identity

Question:
"What is this artifact?"

Characteristics:

- Exactly one
- Represents dominant purpose
- Should remain relatively stable

### 2. Capabilities

Question:
"What can this artifact do?"

Characteristics:

- Zero or more
- Functional
- Independent of identity
- Controlled vocabulary

### 3. Composition

Question:
"How does it work?"

Contains:

- Framework Grammar
- Primitive Cognitive Operations

Only populated when reasoning structure exists.

### 4. Domain

Question:
"Where is it applied?"

One or more domains.

## Controlled Capability Vocabulary

- Reasoning
- Assessment
- Diagnosis
- Decision Support
- Planning
- Execution
- Governance
- Learning
- Representation
- Modeling
- Communication
- Prediction
- Innovation
- Classification
- Measurement
- Compliance
- Documentation
- Coordination
- Visualization
- Risk Analysis
- Optimization
- Simulation
- Evaluation
- Monitoring
- Traceability
- Alignment
- Extensibility

Future additions require evidence.

Avoid synonyms.

## Measurement Principles

- Characterize before classifying.
- Separate identity from capabilities.
- Separate structural properties from functional properties.
- Record uncertainty explicitly.
- Prefer evidence over intuition.
- Use categorical confidence only.
- Maintain traceability to observations.

## Measurement Stability

Once an experimental measurement instrument is adopted for a study, changes should be minimized.

If improvements become necessary:

- Document rationale.
- Version the instrument.
- Preserve historical compatibility.
- Never silently reinterpret previous observations.
