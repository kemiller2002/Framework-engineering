# Framework Taxonomy

Status: Working draft

Framework Taxonomy is a research workstream within Framework Engineering.

## Purpose

Test FE-002, the hypothesis that many domain-specific frameworks are variations of a smaller number of reusable framework grammars.

Framework Taxonomy depends on Knowledge Artifact Taxonomy.

Before classifying a named artifact as a framework, first determine whether it is actually a framework, model, notation, standard, methodology, process, taxonomy, ontology, pattern, reference architecture, or tool.

## Core Method

- collect frameworks
- reduce them to primary question, grammar, output, failure mode
- cluster by grammar rather than domain
- name archetypes only after patterns emerge
- avoid forcing frameworks into predefined categories
