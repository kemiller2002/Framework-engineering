# Operational Definitions

Status: Working draft

## Purpose

Operational definitions support consistent artifact characterization and classification.

These definitions are working rules for FE-008, not final doctrine.

FEMS-1 is the current measurement standard for applying these definitions.

Identity answers:
What is this?

Capabilities answer:
What can it do?

Composition answers:
How does it work?

Domain answers:
Where is it applied?

## Artifact Types

### Framework

Primary signal:
Guides reasoning through reusable concepts, relationships, boundaries, and outputs.

### Model

Primary signal:
Represents some aspect of reality through abstraction.

### Methodology

Primary signal:
Describes how work should be performed in repeatable practice.

### Process

Primary signal:
Prescribes a sequence or flow of activity.

### Taxonomy

Primary signal:
Organizes things into categories.

### Ontology

Primary signal:
Defines concepts and their relationships as semantic structure.

### Standard

Primary signal:
Establishes requirements, conformance rules, or consistency constraints.

### Notation

Primary signal:
Represents information through syntax, symbols, or diagrammatic conventions.

### Pattern

Primary signal:
Captures a recurring successful solution or design move.

### Reference Architecture

Primary signal:
Provides reusable structural guidance for implementation.

### Tool

Primary signal:
Performs work or assists execution directly.

## Boundary Rules

- If execution capability is primary, treat the artifact first as a tool.
- If conformance or requirement-setting is primary, treat it first as a standard.
- If syntax or diagramming dominates meaning, treat it first as a notation.
- If category structure dominates, treat it first as a taxonomy.
- If semantic relationship definition dominates, treat it first as an ontology.
- If representation of reality dominates, treat it first as a model or reference architecture.
- If ordered practice dominates, treat it first as a methodology or process.
- If reusable reasoning structure dominates, framework classification becomes appropriate.
- If the artifact mainly captures a recurring solution, pattern classification becomes appropriate.

## Classification Notes

- Identity should be determined by dominant purpose.
- Identity should contain exactly one value.
- Capabilities may overlap across identities.
- Ambiguous artifacts should remain provisional.
- Framework grammar should be recorded only when framework characteristics are present.
- Primitive composition should be recorded only when reasoning structure exists.
