# Knowledge Artifact Taxonomy

Status: Working draft

## Purpose

Define the broader family of structured knowledge artifacts so Framework Engineering can clearly identify what is and is not a framework.

Framework Engineering applies primarily to frameworks, but frameworks often overlap with models, methodologies, processes, standards, notations, taxonomies, ontologies, patterns, reference architectures, and tools.

This taxonomy is provisional and evidence-driven.

## FE-008 Study Infrastructure

FE-008 adds research infrastructure for characterizing knowledge artifacts before classification.

FEMS-1 is the current measurement standard for this work.

Key rules:

- characterize first, classify second
- separate identity from capabilities
- include framework grammar only when framework characteristics are present
- include primitive composition only when reasoning structure exists
- keep all classifications provisional
- use categorical confidence only
- do not use numeric scores
