# Framework Engineering Cube

Status: Working draft

## Purpose

The Framework Engineering Cube is a provisional way to think about artifact characterization across multiple dimensions without collapsing them into a single score.

FEMS-1 uses this structure as the current FE-008 measurement model.

Identity

↓

Capabilities

↓

Composition

↓

Domain

## Dimensions

### Identity

What is this artifact by dominant purpose?

Exactly one.

### Capabilities

What functions can this artifact perform?

Zero or more.

### Composition

How does it work?

Composition consists of:

- Framework Grammar
- Primitive Cognitive Operations

### Domain

Where is it applied?

One or more domains.

## Use

- Capabilities may overlap across identities.
- The cube is for characterization first, not final classification.
- Framework grammar should be considered only when framework characteristics are present.
- Primitive composition should be considered only when reasoning structure exists.
- Outputs remain provisional.

## Interpretation Notes

- Identity should remain distinct from capability overlap.
- Composition should be analyzed only after identity and capabilities are recorded.
- Mixed profiles should be recorded as provisional without conflating identity and capabilities.
