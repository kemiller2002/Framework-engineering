# Artifact Decision Tree

Status: Working draft

## Purpose

This decision tree supports provisional identity classification after characterization is complete.

Characterize first, classify second.

Use FEMS-1 dimensions before applying the tree.

## Decision Order

```text
tool
↓
standard
↓
notation
↓
taxonomy
↓
ontology
↓
model / reference architecture
↓
methodology / process
↓
framework
↓
pattern
```

## Use Rules

- Start with the dominant observed function, not the most familiar label.
- Stop at the first level that clearly explains the artifact's identity.
- Record capabilities separately from identity.
- Do not assign framework grammar unless framework characteristics are present.
- Do not assign primitive composition unless a reasoning structure exists.
- Preserve unresolved ambiguity in notes.

## Decision Prompts

### Tool

Does the artifact primarily execute work or provide direct operational capability?

### Standard

Does the artifact primarily establish requirements, conformance, or mandated consistency?

### Notation

Does the artifact primarily provide a syntax or representational convention?

### Taxonomy

Does the artifact primarily classify things into categories?

### Ontology

Does the artifact primarily define concepts and semantic relationships?

### Model / Reference Architecture

Does the artifact primarily represent reality or provide reusable structural guidance?

### Methodology / Process

Does the artifact primarily prescribe how work should be performed or sequenced?

### Framework

Does the artifact primarily guide reasoning through reusable concepts, relationships, boundaries, and outputs?

### Pattern

Does the artifact primarily capture a recurring solution or design move?
